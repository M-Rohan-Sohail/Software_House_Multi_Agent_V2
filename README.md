# 🚀 Multi-Agent Software House V2:
**An Autonomous, State-Aware Development Ecosystem Powered by LangGraph & Groq**

## 📖 Project Vision
The **Multi-Agent Software House** is an advanced AI orchestration framework that goes beyond simple code generation. It implements a modular "Software Development Life Cycle" (SDLC) where specialized AI agents collaborate through a centralized state to plan, execute, and refine software projects.

---

## 🧠 Core Architecture & Workflow

The system is built as a **Stateful Graph**, ensuring that every agent has access to the full context of the project, including task history, generated code, and user requirements.

### 1. Request Orchestration (The Manager)
The `manager_agent` acts as the "Client Relations Manager".
* **Intent Classification**: It analyzes the user's prompt to determine the required expertise (Web, Chatbot, Agentic AI, or simple File Editing).
* **Dynamic Routing**: It updates the `next_agent` state to trigger the correct workflow path.

### 2. Task Decomposition (The Planner)
The `planner_agent` serves as the "System Architect".
* **Atomic Subtasks**: It breaks down high-level requests into a list of specific, actionable steps.
* **JSON Blueprinting**: It outputs a structured JSON array defining the agent responsible, the specific task, the target filename, and whether the task is a "New Build" or an "Edit".

### 3. Loop Execution (The Executor)
The `executor_agent` is the project's "Technical Lead".
* **Sequential Processing**: It iterates through the planner's list, invoking specialized developers for each step.
* **Automated I/O**: It handles the physical creation and modification of files on the local disk based on agent output.

---

## 🤖 The Developer Departments

Each developer agent utilizes a **Strict Two-Phase Prompting Strategy** to ensure high-quality code and minimize hallucinations.

### 🌐 Web Development Team
* **Focus**: Full-stack web applications and APIs.
* **Default Stack**: Python/Flask backends with responsive HTML/CSS/JS frontends.

### 🤖 Agentic AI Team
* **Focus**: Autonomous agents and Retrieval-Augmented Generation (RAG).
* **Default Stack**: LangChain, LangGraph, and FAISS vector databases.

### 💬 Conversational AI Team
* **Focus**: Chatbots and advanced dialogue management.
* **Default Stack**: LangChain-based conversational chains and document loaders (PyPDF).

### 🛠️ File Intelligence Team (The Editor)
* **Focus**: Surgical modifications to existing codebases.
* **Function**: It reads existing code and applies precise logic updates while preserving the original structure and comments.

## ⚙️ Technical Specifications

### **State Management (`agentstate`)**
The shared memory object tracks the following variables throughout the graph's lifecycle:
* `task`: The current active subtask description.
* `messages`: Full thread history of user/assistant interactions.
* `generated_code`: Buffer for the latest code block generated by a sub-agent.
* `filename`: The specific file currently being targeted for creation or modification.

### **Inference Details**
* **Model**: `openai/gpt-oss-120b` (via Groq).
* **Utilities**: A custom `clean_code` function using Regular Expressions ensures that Markdown artifacts are stripped before files are saved to the system.
